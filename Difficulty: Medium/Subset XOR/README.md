<h2><a href="https://www.geeksforgeeks.org/problems/subset-xor--175953/1">Subset XOR</a></h2><h3>Difficulty Level : Difficulty: Medium</h3><hr><div class="problems_problem_content__Xm_eO"><p dir="ltr" style="box-sizing: border-box; margin: 0px 0px 10px; padding: 0px; border: 0px; font-size: 18px; vertical-align: baseline; color: #273239; font-family: Nunito, sans-serif; letter-spacing: 0.162px; background-color: #ffffff;"><span style="box-sizing: border-box; margin: 0px; padding: 0px; border: 0px; vertical-align: baseline;">Given an positive integer&nbsp;</span><strong style="box-sizing: border-box; margin: 0px; padding: 0px; border: 0px; vertical-align: baseline;"><strong style="box-sizing: border-box; margin: 0px; padding: 0px; border: 0px; vertical-align: baseline;">n</strong></strong><span style="box-sizing: border-box; margin: 0px; padding: 0px; border: 0px; vertical-align: baseline;">, find a subset of numbers from 1 to n (inclusive), where each number can be used at most once, such that:</span></p>
<ul>
<li dir="ltr" style="box-sizing: border-box; border: 0px; font-size: 18px; vertical-align: baseline; color: #273239; font-family: Nunito, sans-serif; letter-spacing: 0.162px; background-color: #ffffff;"><span style="box-sizing: border-box; margin: 0px; padding: 0px; border: 0px; vertical-align: baseline;">The XOR of all elements in the subset is exactly n.</span></li>
<li dir="ltr" style="box-sizing: border-box; border: 0px; font-size: 18px; vertical-align: baseline; color: #273239; font-family: Nunito, sans-serif; letter-spacing: 0.162px; background-color: #ffffff;"><span style="box-sizing: border-box; margin: 0px; padding: 0px; border: 0px; vertical-align: baseline;">The size of the subset is as large as possible.</span></li>
<li dir="ltr" style="box-sizing: border-box; border: 0px; font-size: 18px; vertical-align: baseline; color: #273239; font-family: Nunito, sans-serif; letter-spacing: 0.162px; background-color: #ffffff;"><span style="box-sizing: border-box; margin: 0px; padding: 0px; border: 0px; vertical-align: baseline;">If multiple such subsets exist, choose the <strong>lexicographically smallest</strong> one.</span></li>
</ul>
<p><span style="box-sizing: border-box; margin: 0px; padding: 0px; border: 0px; vertical-align: baseline;"><strong data-start="65" data-end="91">Lexicographical Order : </strong>A subset <strong>A[]</strong> is lexicographically smaller than subset&nbsp;<strong>B[]</strong> if at the first index where they differ, <strong><code data-start="195" data-end="208">A[i] &lt; B[i]</code></strong> (based on character ASCII/Unicode values).<br data-start="251" data-end="254">If all elements match but one subset ends earlier, the shorter subset&nbsp;is considered smaller.</span></p>
<p><span style="font-size: 18px;"><strong>Examples:</strong></span></p>
<pre><span style="font-size: 18px;"><strong>Input: </strong>n = 4<br><strong>Output: </strong>[1, 2, 3, 4]</span><span style="font-size: 18px;"><br><strong>Explanation:</strong> We choose all the elements from 1 to 4. Its xor value is equal to n. This is the maximum possible size of the subset.</span></pre>
<pre><span style="font-size: 18px;"><strong>Input: </strong>n = 3<br><strong>Output:</strong> [1, 2]<br><strong>Explanation:</strong> 1 ^ 2 = 3, This is the smallest lexicographical answer possible with maximum size of subset i.e 2.</span></pre>
<p><span style="font-size: 14pt;"><strong>Constraints:<br></strong></span><span style="font-size: 14pt;">1 ≤ n ≤ 10<sup>5</sup></span></p></div><br><p><span style=font-size:18px><strong>Topic Tags : </strong><br><code>Bit Magic</code>&nbsp;<code>implementation</code>&nbsp;
