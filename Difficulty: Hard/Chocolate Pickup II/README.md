<h2><a href="https://www.geeksforgeeks.org/problems/chocolate-pickup-ii/1">Chocolate Pickup II</a></h2><h3>Difficulty Level : Difficulty: Hard</h3><hr><div class="problems_problem_content__Xm_eO"><p data-pm-slice="0 0 []"><span style="font-size: 14pt; font-family: 'andale mono', monospace;">You are given a square matrix <strong>mat[][]</strong> of size <strong>n × n</strong>, where each cell represents either a blocked cell or a cell containing some chocolates. If mat[i][j] equals <strong>-1</strong>, then the cell is <strong>blocked</strong> and cannot be visited. Otherwise, mat[i][j] represents the number of chocolates present in that cell.<br></span><span style="font-size: 14pt; font-family: 'andale mono', monospace;">The task is to determine the <strong>maximum number of chocolates</strong> a robot can collected by starting from the top-left cell (0, 0), moving to the bottom-right cell (n-1, n-1), and then returning back to (0, 0). <br></span><span style="font-size: 14pt; font-family: 'andale mono', monospace;">While moving from (0, 0) to (n-1, n-1), the robot can move only in the <strong>right</strong> (i, j+1) or <strong>downward</strong> (i+1, j) directions. On the return journey from (n-1, n-1) to (0, 0), it can move only in the <strong>left</strong> (i, j-1) or <strong>upward</strong> (i-1, j) directions.</span></p>
<p data-pm-slice="0 0 []"><span style="font-size: 14pt; font-family: 'andale mono', monospace;"><strong>Note: </strong>After collecting chocolates from a cell (i, j), that cell becomes empty, meaning <strong>mat[i][j]</strong> becomes <strong>0</strong> for next visit. If there is no valid path from (0, 0) to (n-1, n-1) or for the return trip, the result should be<strong> 0</strong>.</span></p>
<p><span style="font-size: 14pt; font-family: 'andale mono', monospace;"><strong>Example:</strong></span></p>
<pre><span style="font-size: 14pt; font-family: 'andale mono', monospace;"><strong>Input: </strong><span style="font-size: 18.6667px; white-space: normal;">mat[][]</span> = [[0, 1, -1], <br>                [1, 1, -1], <br>                [1, 1, 2]]
<strong>Output: </strong><span style="color: #273239; letter-spacing: 0.162px; text-align: justify; text-wrap-mode: wrap; background-color: #f9f9f9;">7</span>
<strong>Explanation:<br>  <img src="https://media.geeksforgeeks.org/img-practice/prod/addEditProblem/914224/Web/Other/blobid0_1762493171.jpg" width="245" height="217"><br></strong>One of the optimal paths is to move from (0,0) -&gt; (1,0) -&gt; (2,0) -&gt; (2,1) -&gt; (2,2) while going forward, and then from (2,2) -&gt; (2,1) -&gt; (1,1) -&gt; (0,1) -&gt; (0,0) while coming back. The total number of chocolates collected is 7.</span></pre>
<pre><span style="font-size: 14pt; font-family: 'andale mono', monospace;"><strong>Input: </strong><span style="font-size: 18.6667px; white-space: normal;">mat[][]</span> = <span style="color: #273239; letter-spacing: 0.162px; text-wrap-mode: wrap; background-color: #f9f9f9;">[[1, 1, 0], <br>               [1, 1, 1], <br>               [0, 1, 1]]</span>
<strong>Output: </strong><span style="color: #273239; letter-spacing: 0.162px; text-align: justify; text-wrap-mode: wrap; background-color: #f9f9f9;">7</span>
<strong>Explanation:<br>  <img src="https://media.geeksforgeeks.org/img-practice/prod/addEditProblem/914224/Web/Other/blobid1_1762493245.jpg" width="245" height="216"><br></strong>One of the optimal paths is to move from <code data-start="112" data-end="119">(0,0)</code> -&gt;<code data-start="122" data-end="129"> (1,0)</code> -&gt; <code data-start="132" data-end="139">(2,0)</code> -&gt; <code data-start="142" data-end="149">(2,1)</code> -&gt; <code data-start="152" data-end="159">(2,2)</code> while going forward, and then from <code data-start="195" data-end="202">(2,2)</code> -&gt; <code data-start="205" data-end="212">(1,2)</code> -&gt; <code data-start="215" data-end="222">(1,1)</code> -&gt; <code data-start="225" data-end="232">(0,1)</code> -&gt; <code data-start="235" data-end="242">(0,0)</code> while coming back. The total number of chocolates collected is<strong> </strong>7.</span></pre>
<pre><span style="font-family: 'andale mono', monospace; font-size: 14pt;"><strong>Input: </strong><span style="font-size: 18.6667px; white-space: normal;">mat[][]</span> = [[1, 0, -1],<br>                [2, -1, -1],<br>                [1, -1, 3]]
<strong>Output: </strong>0<strong>
Explanation:<br>  <img src="https://media.geeksforgeeks.org/img-practice/prod/addEditProblem/914224/Web/Other/blobid4_1761906108.webp" width="255" height="225"><br></strong>It is impossible to reach the bottom-right cell (2,2) from (0,0) because every route is blocked. <p><span style="font-size: 14pt; font-family: 'andale mono', monospace;"><strong>Constraint:</strong></span><br><span style="font-size: 14pt; font-family: 'andale mono', monospace;">1&nbsp;<span style="color: #1e2229; background-color: #ffffff;">≤</span>&nbsp;n&nbsp;<span style="color: #1e2229; background-color: #ffffff;">≤ 50</span><br>-1&nbsp;<span style="color: #1e2229; background-color: #ffffff;">≤</span>&nbsp;</span><span style="font-size: 18.6667px; font-family: 'andale mono', monospace;">mat</span><span style="font-family: 'andale mono', monospace; font-size: 14pt;">[i][j]&nbsp;</span><span style="color: #1e2229; background-color: #ffffff;">≤</span><span style="font-family: 'andale mono', monospace; font-size: 14pt;">&nbsp;100</span></p></div><br><p><span style=font-size:18px><strong>Topic Tags : </strong><br><code>Dynamic Programming</code>&nbsp;<code>Recursion</code>&nbsp;
